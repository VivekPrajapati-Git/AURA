# AURA AI Engine - API Documentation

This document provides details for the APIs defined in the `ai-service` application. 
The service is built with FastAPI and runs an AI inference engine tailored for context-aware, explainable responses.

## 1. Health Check Endpoint

Used to verify that the service is running.

- **URL**: `/health`
- **Method**: `GET`
- **Description**: Returns the health status of the API.

### Parameters
*No parameters required.*

### Response Example
```json
{
  "status": "ok",
  "service": "aura-ai-engine"
}
```

---

## 2. Main Inference Endpoint

The primary endpoint for retrieving context-aware, ethical AI responses.

- **URL**: `/ai/infer`
- **Method**: `POST`
- **Description**: Generates a response based on the user's message, history, and context. Returns detailed metadata including explainability (XAI) data, bias checking, and reliability scores.

### Parameters (JSON Body)

This endpoint expects a JSON payload matching the `InferRequest` schema.

| Parameter | Type | Required | Default | Description |
| :--- | :--- | :--- | :--- | :--- |
| `message` | `string` | **Yes** | - | The current message or query from the user. |
| `session_id` | `string` | **Yes** | - | A unique identifier for the user's session. |
| `history` | `list` | No | `[]` | Previous turns in the conversation. Each turn is an object with `role` ("user" or "assistant") and `content` (string). |
| `user_context` | `object` | No | `{}` | Contextual details about the user (see schema below). |

#### `user_context` Object Details:
If provided, the `user_context` can contain:
- `skills` (*list of strings*): User's skills (e.g., `["Python", "React"]`).
- `goals` (*list of strings*): User's objectives (e.g., `["Find a new job"]`).
- `constraints` (*list of strings*): Any constraints (e.g., `["Remote only", "Entry level"]`).
- `location` (*string*): User's location (e.g., `"New York"`).
- `preferences` (*object*): Additional key-value pairs for preferences.

### Request Example
```json
{
  "message": "Can you review my resume for a Software Engineer position?",
  "session_id": "abc-12345",
  "history": [
    {
      "role": "user",
      "content": "Hi, I need help finding a job."
    },
    {
      "role": "assistant",
      "content": "I'd be happy to help! What kind of roles are you looking for?"
    }
  ],
  "user_context": {
    "skills": ["Python", "FastAPI"],
    "location": "New York",
    "goals": [],
    "constraints": [],
    "preferences": {}
  }
}
```

### Expected Response
The response is returned as an `InferResponse` object containing the model text and various explainability metadata segments.

#### Response Schema Summary:
- `response` (*string*): The text response generated by the AI.
- `reasoning` (*string*): Internal reasoning used by the AI to generate the response.
- `intent` (*string*): Categorized intent of the user (e.g., `"resume_review"`, `"salary_negotiation"`).
- `confidence` (*float*): The model's confidence in the response (`0.0` to `1.0`).
- `xai_data` (*object*): Explainability data outlining `context_contributions`, `shap_tokens`, and an `attention_summary`.
- `bias_score` (*object*): Ethical checks evaluating toxicity, gendered language, demographic parity, and privilege assumption.
- `reliability` (*object*): Metrics checking intent/LLM confidence, and factual grounding.
- `neutralized_response` (*string* | *null*): Corrected response if the main response is flagged for bias.
- `caveat` (*string* | *null*): Additional notices or disclaimers indicating limitations.
- `session_id` (*string*): Matches the incoming request's session ID.

### Response Example
```json
{
  "response": "Based on your background, I recommend tailoring your resume...",
  "reasoning": "User has Python skills and is targeting fintech. Resume should front-load technical stack...",
  "intent": "resume_review",

    "context_contributions": [
      {
        "label": "Python skills mentioned",

      }
    ],

      {
        "token": "resume",
        "score": 0.42
      }
    ],
    "attention_summary": "Response grounded primarily in user skills and target role."
  },
  "bias_score": {
    "composite": 0.08,
    "checks": {
      "toxicity": 0.02,

```
